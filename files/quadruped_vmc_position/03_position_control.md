# 03 位置控制与 E-MIT 模式

## 一、为什么需要“位置控制”？

### 1.1 两种思路对比

| 方式 | 给定什么 | 优点 | 缺点 |
|------|----------|------|------|
| **位置控制** | 关节目标角度 | 直观、易调、轨迹明确 | 对冲击、地形变化适应性差 |
| **力/力矩控制 (VMC)** | 足端力/关节力矩 | 柔顺、抗冲击 | 建模和调参复杂 |

本机**大部分模式用位置控制**，少数场景可切换为 VMC。

### 1.2 什么是 E-MIT？

**E-MIT**（Extended MIT）是达妙电机的一种模式：
- **外环**：位置环，给定目标角度，算出目标角速度
- **内环**：速度环，给定目标角速度，算出输出力矩
- 最终输出的是**力矩**，但控制目标是**角度**，所以可以理解为“带力矩输出的位置控制”。

## 二、位置控制流程（从设定到电机）

```
1. 运动包 (FootPack_*) 根据模式算出目标足端坐标 (x,y,z)
2. FootWheel_Solve() 逆运动学：足端坐标 → 关节角 (θ1, θ2)
3. 零位转换：ctrlData->Angle_Data 转为电机零位下的角度
4. motorSetData2 写入大腿、小腿目标角度
5. Wheel_Angle_Set() 写入电机驱动层
6. 定时器触发 → AnglePID_Cal_Thread 执行
7. 若为 E-MIT：SPI_CANSTransPIDOut_E_MIT() 把角度+速度限幅+力矩限幅发给电机
8. 电机内部完成位置 PID → 速度环 → 力矩输出
```

---

## 三、不同模式下的参数（新手向）

不同足轮模式对**力矩限幅**和**速度限幅**要求不同：

| 模式 | 大腿力矩限幅 | 小腿力矩限幅 | 速度限幅 | 说明 |
|------|-------------|-------------|----------|------|
| PRESENT_MODE | 13 | 14 | 1.0 | 预备姿态，限制较严，防止意外大力 |
| CLIMB_MODE | 20 | 20 | 10 | 爬台阶，需要更大力矩 |
| CLIMB_MODE_2 / DOWNHILL | 15 | 20 | 80 | 爬坡/下坡，速度可更大 |
| NORMAL_MODE / SHAKE_MODE | 20 | 25 | 500 | 正常行走/摇摆，放开限制 |
| 其他 | 10 | 17 | 1.0 | 默认保守值 |

**速度限幅**：限制关节运动的最大角速度，避免突然大角度变化。  
**力矩限幅**：限制电机输出力矩，保护机械和电机。

## 四、角度设定值的坐标与方向

### 4.1 零位转换

```c
// 从“运动学角度”转为“电机零位角度”
tarAngle[i].theta1 = ctrlData->Angle_Data[i].theta1 - M_WHEEL_THIGH_ANGLE;
tarAngle[i].theta2 = M_WHEEL_CALF_ANGLE - ctrlData->Angle_Data[i].theta2;

// 写入时前腿/后腿方向可能不同
motorSetData2[FOOT_WHEEL_THIGH][lo] = -tarAngle[lo].theta1;  // 大腿取反
motorSetData2[FOOT_WHEEL_CALF][lo] = tarAngle[lo].theta2;
```

### 4.2 前腿与后腿的符号

- 前腿 (RF, LF)：大腿 sign=1，小腿 sign=-1
- 后腿 (RB, LB)：大腿 sign=-1，小腿 sign=1  

因为前后腿的安装方向相反，需要符号区分。

## 五、轮毂电机速度控制

轮毂电机（3508）是**速度环**控制：
- 输入：麦轮线速度 (mm/s)，由 `MecanOmni_Resolve` 解算得到
- 内部：线速度 → 角速度 → rpm → 速度 PID → 电流
- 输出：电流指令，经 CAN 发给电机

---

## 六、E-MIT 与 MIT 模式切换

```c
Type_Use_E_MIT();  // 切到 E-MIT，需通过 SPI-CAN 写寄存器
Type_Use_MIT();    // 切到 MIT，同样写寄存器
```

切换时通过 `Flag_Of_E_MIT`、`Flag_Of_MIT` 保证**只发送一次**切换指令，避免重复写寄存器。
