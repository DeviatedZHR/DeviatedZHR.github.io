## 任务1 配Bsp
RT-Thread 是一个嵌入式实时操作系统（RTOS），它为各种硬件平台提供了良好的支持。HAL（Hardware Abstraction Layer，硬件抽象层）是操作系统或应用程序与硬件之间的一个软件层，它的主要目的是将硬件相关的细节与上层软件隔离开来，从而使得操作系统或应用可以更加独立于具体的硬件平台运行。这样做的好处是可以提高系统的可移植性，并简化开发过程。

在 RT-Thread 中，HAL 层是其实现的一部分，它提供了一种方法来处理不同硬件平台上的设备驱动程序和其他底层操作。RT-Thread 的 HAL 实现通常包括了对常见外设的初始化和操作，比如定时器、串口、GPIO（通用输入输出端口）、SPI（串行外设接口）等，这些都通过一个统一的接口来实现。

具体来说，在 RT-Thread 中使用 HAL 层有以下好处：

1. **简化硬件编程**：开发者可以通过标准的 API 调用来控制硬件，而不需要关心硬件的具体实现细节。
2. **增强代码的可移植性**：如果更换了硬件平台，只需要重新配置或编写相应的 HAL 层代码，而不需要改动上层的应用逻辑。
3. **隔离硬件依赖**：HAL 层作为硬件和操作系统之间的桥梁，可以让操作系统的核心部分保持相对纯净，不包含任何特定硬件的细节。

值得注意的是，除了 RT-Thread 自带的 HAL 层之外，一些硬件厂商也会为他们的产品提供专门的 HAL 库，以方便开发者快速地在其硬件平台上部署 RT-Thread 或其他操作系统。此外，像 STM32Cube HAL 库就是由 STMicroelectronics 提供的用于 STM32 微控制器的 HAL，可以与 RT-Thread 结合使用来加速开发过程。

如果是**自收自发**，则采用**循环模式**；如果是**主机发从机或外设**，选择**正常模式**即可。由于我们需要驱动电机，则选择正常模式。

### CAN
根据官方文档：[RT-Thread 文档中心](https://www.rt-thread.org/document/site/#/rt-thread-version/rt-thread-standard/programming-manual/device/can/can?id=can-%e8%ae%be%e5%a4%87%e4%bd%bf%e7%94%a8%e7%a4%ba%e4%be%8b)

can的使用都是基于一个新线程，在线程里面进行读取或写入数据。

根据这个特性，可以写出CAN的配置函数。



首先需要写出can设备名，以can1为例（H7板为fdcan1）


```c
/* can初始化 */
void can_init(void)
{
		/* 查找 CAN 设备 */
		can_dev = rt_device_find(CAN_DEV_NAME);
		/* 以中断接收及发送方式打开 CAN 设备 */
		rt_device_open(can_dev, RT_DEVICE_FLAG_INT_TX | RT_DEVICE_FLAG_INT_RX);
		/* 设置 CAN 通信的波特率为 1Mbit/s*/
		rt_device_control(can_dev, RT_CAN_CMD_SET_BAUD, (void *)CAN1MBaud);
		/* 设置 CAN 的工作模式为正常工作模式 */
		rt_device_control(can_dev, RT_CAN_CMD_SET_MODE, (void *)RT_CAN_MODE_NORMAL);
}
```

```c
/* 接收例程 */
/* 接收数据回调函数 */
static rt_err_t can_rx_call(rt_device_t dev, rt_size_t size)
{
    /* CAN 接收到数据后产生中断，调用此回调函数，然后发送接收信号量 */
    rt_sem_release(&rx_sem);

    return RT_EOK;
}

static void can_rx_thread(void *parameter)
{
    struct rt_can_msg rxmsg = {0};

    /* 设置接收回调函数 */
    rt_device_set_rx_indicate(can_dev, can_rx_call);

    while (1)
    {
        /* hdr 值为 - 1，表示直接从 uselist 链表读取数据 */
        rxmsg.hdr = -1;
        /* 阻塞等待接收信号量 */
        rt_sem_take(&rx_sem, RT_WAITING_FOREVER);
        /* 从 CAN 读取一帧数据 */
        rt_device_read(can_dev, 0, &rxmsg, sizeof(rxmsg));
				
		NowValue.angle = ( (int16_t)rxmsg.data[0] << 8 ) | rxmsg.data[1];
		NowValue.rpm = ( (int16_t)rxmsg.data[2] << 8 ) | rxmsg.data[3];
		NowValue.moment_current = ( (int16_t)rxmsg.data[4] << 8 ) | rxmsg.data[5];
		NowValue.temperature = (int16_t)rxmsg.data[6];

		/* 打印数据 ID 及内容 */
		//rt_kprintf("ID:%x\tmessege:\t%d,%d,%d,%d\n", rxmsg.id, NowValue.angle,NowValue.rpm,NowValue.moment_current,NowValue.temperature);
		rt_thread_mdelay(10);
    }
}


void can_sample_receive(void)
{
    rt_thread_t thread;

    /* 初始化 CAN 接收信号量 */
    rt_sem_init(&rx_sem, "rx_sem", 8, RT_IPC_FLAG_FIFO);

    /* 创建数据接收线程 */
    thread = rt_thread_create("can_rx", can_rx_thread, RT_NULL, 1024, 5, 50);
		
    /* 开启线程 */
		rt_thread_startup(thread);
}
```

```c
/* 发送例程 */
static void can_tx_thread(void *parameter)
{
    struct rt_can_msg msg = {0};
    msg.id = 0x1FF;              /* ID 为 0x1FF */
    msg.ide = RT_CAN_STDID;     /* 标准格式 */
    msg.rtr = RT_CAN_DTR;       /* 数据帧 */
    msg.len = 8;                /* 数据长度为 8 */

    while(1)
    {
		/* 待发送的 8 字节数据 */
		//rt_memset(msg.data,0,sizeof(msg.data));
		msg.data[0] = 0x00;
		msg.data[1] = 0x00;
		msg.data[2] = (int8_t)(int16_t)(send_data >> 8);
		msg.data[3] = (int8_t)(int16_t)send_data;
		msg.data[4] = 0x00;
		msg.data[5] = 0x00;
		msg.data[6] = 0x00;
		msg.data[7] = 0x00;
				
        /* 发送一帧 CAN 数据 */
        rt_device_write(can_dev, 0, &msg, sizeof(msg));
        rt_thread_mdelay(10);
    }
}
```

### 定时器
根据官方文档：[RT-Thread 文档中心](https://www.rt-thread.org/document/site/#/rt-thread-version/rt-thread-standard/programming-manual/timer/timer?id=%e5%ae%9a%e6%97%b6%e5%99%a8%e5%ba%94%e7%94%a8%e7%a4%ba%e4%be%8b)

与CAN的配置类似，定时器也是添加一个线程来实现。



首先需要启动定时器

```c
static struct rt_timer timer1;
```



然后检查清理之前的定时器

```c
/* 检查，清理上次的定时器 */
static void check_timer_exist(void)
{
   if (rt_object_find("timer1", RT_Object_Class_Timer) != RT_NULL)
   {
       rt_timer_detach(&timer1);
   }
}
```



接下来需要写定时器的**超时函数**，这个相当于**定时器中断**，但是比它更为高级。

```c
/* 定时器 1 超时函数 */
static void timeout1(void* parameter)
{
	...
}
```



这样就可以写初始化函数了，这里包括创建关于定时器的线程。

```c
/* 定时器初始化 */
void Timer_init(void)
{
	/* 检查，清理上次的定时器 */
    check_timer_exist();
    /* 初始化定时器 */
    rt_timer_init(&timer1, "timer1",  /* 定时器名字是 timer1 */
                timeout1, /* 超时时回调的处理函数 */
                RT_NULL, /* 超时函数的入口参数 */
                10, /* 定时长度，以 OS Tick 为单位，即 10 个 OS Tick */  //这里一个OS Tick为 1ms
                RT_TIMER_FLAG_PERIODIC); /* 周期性定时器 */
	
	/* 启动定时器 */
    rt_timer_start(&timer1);
}
```



这样在主函数中就可以直接调用了。

```c
/* timer */
Timer_init();
/* timer */
```

```c
//转速换算电压
float F(float x)
{
	return x > 0 ? (70.13783072447707f * x + 349.06150436348616f) : x == 0 ? 0 : (70.13783072447707f * x - 349.06150436348616f);
}
```

```c
// 使用示例     AngleSpeedCtr_Init();
// 备注信息
//-------------------------------------------------------------------------------------------------------------------
void AngleSpeed_Ctr(float angle_u,int16_t now_rpm)
{
		//目标值为角度环的输出值
		AngleSpd_Pid.goal = angle_u;
		
		//角度闭环pid，使用位置式
		Location_Pid(&AngleSpd_Pid, now_rpm);
}
```

这样电机就可以任意转动了。



这里需要注意的是：**电机在0度和359度之间会突变**。为了防止该现象，需要对角度环的误差值做一些处理：

```c
//-------------------------------------------------------------------------------------------------------------------
// 函数简介     位置式pid 云台控制使用
// 参数说明     *pid pid名称  actual 实际值
// 返回参数     void
// 使用示例     Location_Pid_For_Angle(Spd_pidL,Speed);
// 备注信息
//-------------------------------------------------------------------------------------------------------------------
void Location_Pid_For_Angle(Lpid_menber *pid, float actual)
{
    float e = actual - pid->goal;
		
		if(e>180)e-=360;
		if(e<-180)e+=360;
		
    //储存累计偏差值
    pid->ei += e * pid->freq;
    //积分限幅
    if(pid->ei > pid->i_limit) pid->ei = pid->i_limit;
    else if(pid->ei < -pid->i_limit) pid->ei = -pid->i_limit;
    //位置式pid
    pid->u = pid->kp*e + pid->kd*(e-pid->e_last)/pid->freq + pid->ki*pid->ei;
    //储存上次偏差值
    pid->e_last = e;

}
```
